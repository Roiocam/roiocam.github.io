Posted by Andy Chen on 2020-03-09

一. Hash表简介

在学习 HashMap 之前,需要先了解一下数据结构HashTable.

  • 哈希表的经典实现是数组+链表
  • 时间复杂度是: O(1) [平均查找,插入,删除时间]
  • 致命缺陷: 哈希碰撞

现代的通讯录就是一个哈希表的数据结构。通讯录使用字母当作索引，一共有26个字母,数组大小为26 => LinkedList[] dict = new LinkedList();

经过哈希算法后，哈希值相同的情况则称为哈希碰撞，也就是哈希结果都为Z，这时候就需要所有结果都为Z的元素存入Z存储的链表中。

与通讯录不同的是，HashTable要求所有数组中元素分布均匀，不允许只有Z姓链表有元素，而其他为空，这样的话，哈希表会退化成链表。

二. JAVA7的HashMap

前言

JAVA7的 HashMap 使用了经典的哈希表实现: 数组+链表,并无优化，不同于JAVA中线程安全的HashTable. HashMap是简单实现且非同步.故HashMap的效率会更高一点.

哈希桶 : 代指数组+链表中的链表，因为哈希桶在数组中的索引是对哈希值的比特取位获得的。

再用通讯录举例,

Z->张三->张三丰

在这里，Z是哈希值，整个”Z->张三->张三丰”则是哈希桶。多个哈希桶构成了HashMap

2. 潜在安全隐患

因为String的hashCode算法十分简陋,非常容易造成哈希碰撞.如Aa,BB,C#三者的hashCode都是2122.

在大量哈希碰撞后,HashMap则会从哈希表退化到链表，导致出现查找性能问题。

由于Tomcat的请求参数使用HashMap存储，使用大量HashCode相同的String,可以对服务器进行DOS攻击，造成服务器生成大量链表，导致查询性能差。
