LeetCode.912 Sort Array

Posted by Andy Chen on 2020-04-04

前言

这一道初级得不能再初级的算法题目,由于排序的方法有很多,根据不同的“时间/空间”的需求,以及开发成本上的考虑,来选择不同的算法.

排序是《算法》此书的第一章，涉及到了许多不同排序算法，通过对不同算法的理解，从而延伸出各种算法设计上的思想，以及在实际应用时的考虑。

排序算法分类

根据排序算法的不同，大致可以分为几类：

  1. 交换类：冒泡排序，快速排序
  2. 选择类：选择排序，堆排序
  3. 插入类：插入排序，Shell(希尔)排序
  4. 合并类：归并排序
  5. 分治类：桶排序，计数排序，Radix（基数）排序

各分类算法

2. 快速排序 | 平均:O(nlogn) 最坏:O(n^2)

快速排序则是经典的排序算法,它虽然归于属交换类，但是它利用了分而治之的思想。其算法思想为从数组中选定“中点”来将数组分为上下半区，其中上半区元素永远大于下半区，在分别对上下半区递归的排序，从而达到排序数组的目的。

二.选择类排序

选择类排序常见算法是选择排序和堆排序，其核心思想是就地比较.

1. 选择排序 | 平均:O(n^2) 最坏:O(n^2)

由于选择排序的时间复杂度为O(n^2)，故其在大型集合上的效率低下，但在某些情况下比复杂算法具有优势。

选择排序以简单性著称,其优点是最坏情况下的交换次数仍为(n-1)

2. 堆排序 | 平均:O(nlogn) 最坏:O(nlogn)

堆排序也是基于比较的排序算法，可以视为选择排序的改进算法，是就地算法(in-place)，但不是稳定的排序。

大多数情况下效率比快速排序更低，但其最坏情况下O(nlogn)优于快速排序。

堆排序首先将列表转换为最大堆，然后重复删除堆的最大元素，将其插入至数组。
