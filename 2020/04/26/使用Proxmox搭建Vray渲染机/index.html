一. 前言

最近朋友创业的工作室因为磁盘空间不足，以及想要搭建一个 CPU 性能更好的服务器用于渲染任务，于是找到了我。

初步的想法是购买多核性能强劲的 AMD Ryzen 系列. 查阅 Vray Benchmark 的性能如下,发现纯 CPU,组服务器的性能更好，且服务器的内存和硬盘价格都低于家用版，故使用了性价比最高的 E5-2678 V3。

三. 安装

1. 内存安装

服务器的内存与家用的内存不一样，服务器由于是双路主板，故内存插槽被分为P1,P2，且服务器插槽必须要求严格的顺序一致，下面来看一个标准的插槽：超微X10DRI-i

其内存顺序必须遵守如下规则:

  1. P1A1
  2. P2A1
  3. P1B1
  4. P2B1
  5. …

即P1,P2对应两个CPU，其内存插槽必须对称，且优先插入蓝色主插槽，若蓝色已满则插入黑色副插槽（识别顺序）

但有例外，如下面的：超微X10DRU-i的插槽，名称改动了很多。

可以看到虽然P1,P2对应两个CPU不变，但后缀的顺序只是标明了（蓝色主插槽，黑色副插槽），但其根本的对称不变，故X10DRU-i的插入顺序应该是：

  1. P1A1
  2. P2H3
  3. P1B1
  4. P2G3
  5. P1C3
  6. P2E1
  7. …

2. 网口安装

服务器网口的安装则较为简单,但对于小白来说,仍需在这里说明一下,避免大家在安装时走入误途.

服务器通常包含多个网口，例如我的X10DRU准系统，则有5个网口，但是其中4个为10G网卡的网口，只有1个是主板自带的，查询官网显示也是如此，该网口在官网上和主板上都标明了为IPMI网口，也就是集成了IPMI微系统的网口，在安装时,通常需要连接IPMI网口和另一个网口,否则会导致在安装Proxmox时，无法访问Proxmox的Web控制面板。

四. IPMI

IPMI是个微型系统，使我们可以远程控制服务器而无需在服务器现场。

1. 连接到IPMI

首先通过网线连接至IPMI网口，然后在路由器上寻找匿名主机的ip地址（在我的TPLINK上显示为匿名主机）。也可以使服务器连接到显示器，然后通过开机界面，查看IPMI的Baseboard Management Controller (BMC) ip地址。

2. 登录IPMI

找到IPMI系统的BMC地址后，我们需要登录至IPMI进行远程控制，BMC的帐号密码一般默认为ADMIN （都是大写）

如上图的ip 1.219，则我们的登录地址为https://192.168.1.219,这里HTTPS是必须的。

3. 控制服务器

五. 安装Proxmox

1. 远程安装系统

通常我们安装个人主机的系统时，都需要我们自己制作一个启动盘，然后插入USB进行系统安装，但是有了IPMI，我们无需制作启动盘，可以通过远程的方式安装系统。（JAVA Web Start版）

2. 挂载ISO至服务器

第一步:点击Virtual Media的Virtual Storage.

第二步:选择CDROM，并且选择ISO FILE，然后点击Open Image选择系统镜像，最后点击Plugin in插入

3. 安装Proxmox

有关Proxmox的安装网上教程都比较丰富，这里就不过多讲解，教程我推荐“知乎上的一篇文章”.

九. Deadline 渲染农场 (非专业)

Deadline 渲染农场的包含:

  1. 渲染节点 Deadline Client
  2. 用于存储管理任务和脚本的仓库 Deadline Repository
  3. 用于存储任务信息的数据库 MongoDB

1. 安装 Repository 和 Database

MongoDB的安装方式有两种,一是通过安装Repository时安装，二是先于Repository安装,在安装 Repository时指定MongoDB连接配置

由于我本地已经有MongoDB，故我采用第二种方法.

首先在Thinkbox官网上下载Repository，我这边选择将Repository安装在Proxmox系统上。一是因为Linux更加稳定，二是服务端无需破解。

文件下载后,直接执行sudo ./DeadlineRepository-1.1.5.1-linux-x64-installer.run

目录之后则是权限，我们选择Full Access,即输入 y 确定

这是提示我们是否安装MongoDB.我们选择[2]

2. 安装 Deadline Client

该程序需要安装在每一个机器，其功能有:渲染任务监听，渲染任务发布，渲染任务领取。

由于我的服务器是用于3DSMAX的VRAY，故我的客户端安装在Windows下，其安装比较简单，在过程中选择 Repository 路径为共享目录中 Repository的安装路径 /mnt/share/public，之后一路确认即可。

客户端安装完成并打开，我们需要为3DMAX安装提交脚本，打开Monitor监听器，在Tools-> Download Submission Script 中选中3dsmax下载3dsmax的任务发布安装程序.

下载完成后有安装程序和脚本两种,

  1. 将脚本拷贝至3dsMax安装路径/MaxScrpit和3dsMax安装路径/startup/script路径下
  2. 执行安装程序安装

注意 Deadline Slave程序是渲染机才打开的程序,非渲染机只需打开Monitor查看任务即可.

3. 渲染机安装 3dsMax 和 Vray

由于Deadline的渲染任务领取是通过脚本来启动3dsMax执行渲染，故仍需安装 3dsMax程序.

注意，3dsMax版本推荐选择英文版,方便Deadline启动3dsMax。

若在执行任务时，发现Deadline找不到 3dsmax，则需要在Monitor中 Tools - SuperMode -> 之后再 Tools - Configure 3dsMax …来重新选择3dsmax.exe路径..

至此,基于Proxmox的多虚拟机系统和Deadline渲染农场安装完毕.

写在最后

关于软件和系统的选择。

在渲染农场软件上，我们并无过多选择，除了AutoDesk的BackBuner之外，只有ThinkBox的Deadline.

为什么选择Proxmox，其实Deadline农场并不需要Proxmox,且即使是KVM也会略微降低性能，也可以为服务器安装Windows时，并通过共享来解决文件共享，以及Repository安装。

选择Proxmox，一方面是因为Proxmox这类虚拟机系统包含了集群方案，不同与ESXi的收费，和不支持集群的虚拟机。

故综合之下选择了Proxmox作为控制台，后续需要再添加一台 1U服务器来提高性能时，使用Proxmox也便于管理，且Proxmox原生支持存储池以及ZFS文件系统，对后续的扩展带来便利…

最后此章只是在初期时的一些思路,为后续的水平扩展留下便利.希望读者能体会其中思想.
