Quartz任务调度笔记

Posted by Andy Chen on 2019-08-23

强大的开源任务调度框架:Quartz

体系架构

三大核心：调度器，触发器，任务

1). 任务(Job)

a.Job

Job接口表示一个任务，要执行的具体内容，此接口只有一个execute方法用于执行任务逻辑。创建任务则必须实现该接口。Job类似Runnable接口

生命周期：Job接口的实例每次在被触发执行execute逻辑时生成，执行完毕后被释放。由垃圾回收器回收。

c.JobDetail

JobDetail用来保存任务的详细信息，一个JobDetail可以有多个Trigger(触发器)，但是一个Trigger只能对应一个JobDetail。

参数名 类型 备注
name String 任务名称
group String 任务所在组，默认为DEFAULT
jobClass Class 任务的实现类
description String 描述
jobDataMap JobDataMap 用于给Job提供数据支持的数据结构
volatility Boolean 重启应用后是否删除任务相关信息
durability Boolean 任务完成后是否依然保留到数据库
shouldRecover Boolean 重启后是否忽略过期任务
JobListerners Set 监听器

2). 触发器(Trigger)

触发器用于触发任务执行，

a.属性

  • JobKey：表示job实例的标识
  • StartTime：触发器的h时间表首次处罚的时间。
  • EndTime：指定触发器不再被触发的时间。

    b.类型

  1. SimpleTrigger：针对一些相对简单的时间触发进行配置使用。比如指定时间内开始在一定时间间隔之内重复触发执行一个任务。
  2. CronTrigget：可以配置更复杂的触发时刻表，基于日历的作业触发器，而不像SimpleTrigger那样精确指定间隔时间，比SimpleTrigger更加常用。（使用Cron表达式）
  3. NthIncludedDayTrigger：是 Quartz 开发团队最新加入到框架中的一个 Trigger。它设计用于在每一间隔类型的第几天执行 Job。例如，你要在每个月的 15 号执行开票的 Job，用 NthIncludedDayTrigger就再合适不过了。

3). 调度器Scheduler

调度器代表一个Quartz的独立运行容器。Trigger和JobDetail可以注册到Scheduler中，两者在Scheduler中拥有各自的组及名称，组及名称是Scheduler查找定位容器中某一对象的依据，Trigger的组及名称必须唯一，JobDetail的组和名称也必须唯一（但可以和Trigger的组和名称相同，因为它们是不同类型的）
