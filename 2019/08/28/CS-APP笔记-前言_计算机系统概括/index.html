一.C程序被编译的过程

编译为可执行文件:root> gcc -o hello hello.c
此时可通过root> ./hello 执行hello程序
上面的过程可分解为:

  1. hello.c(文本,源代码) –>被预处理(cpp):读取头文件并插入到程序文本
  2. hello.i(修改后的文本) –>编译器(ccl)编译:被编译翻译成汇编语言
  3. hello.s(汇编程序) –>被汇编器(as)翻译成机器语言
  4. hello.o(目标程序-二进制) –> 被重定向:被链接到到目标文件，可以被加载到内存
  5. hello(目标程序)

链接错误：链接时，可能因为定义相同名称的两个全局变量等..只有在运行时才会出现的链接错误

二.硬件组成

  1. 总线:贯穿整个系统的管道。被设计成传送定长的字节块(word)
  2. I/O设备:通常为四个I/O设备:用户输入的键盘，鼠标。用户输出的显示器。存储数据的磁盘驱动器。通过控制器/适配器连接总线。
  3. 主存:临时存储器，由动态随机存取存储器(DRAM)芯片组成。从逻辑上说，存储器是一个线性的字节数组，每个字节有唯一的地址（索引）。
  4. 处理器：解释和执行主存中指令的引擎。核心是一个大小为一个字的存储器(寄存器)，成为程序计数器。在任意时刻，PC都会指向主存中某条机器语言指令。执行指令的顺序由指令集架构决定。CPU的一些操作:
    • 加载:从主存复制一个字节到寄存器。
    • 存储:从寄存器复制一个字节到主存
    • 操作:把两个寄存器内存复制的到ALU，ALU对其进行算术运算，并将结果存储到寄存器。
    • 跳转:从指令本身抽取一个字，复制到程序计数器(CPU)中

三.HelloWorld程序的硬件过程

  1. 等待用户输入，shell读取命令到寄存器,寄存器存储到主存中。
  2. 用户输入结束后，shell执行指令从磁盘加载hello文件复制到主存。
  3. 处理器开始读取执行hello程序,再从寄存器复制到显示器。

    1.高速缓存的重要性

    从上面的执行过程中，系统花费了大量时间把信息从一个地方到另一个地方。
  4. 加载时:hello程序从磁盘到主存，数据串”hello world\n”从磁盘到主存
  5. 执行时:hello程序从主存被复制到寄存器，数据串”hello world\n”从主存到显示设备
    由于机械原理，容量大的存储设备运行得更慢。磁盘驱动器的开销比主存大1000倍。对于处理器而言又是100万倍。
    针对这种差异，系统设计者采用了更小更快的存储设备，成为高速缓存存储器(cache)。也就是现在处理器的l1,l2,l3缓存。
    L1,L2缓存是用一种叫做静态随机访问存储器(SRAM)实现的。
    系统可以获得一个很大并且快速的存储器，原因是利用了高速缓存的局部性原理（程序具有访问局部区域里的数据和代码的趋势）。

    2.存储器层次结构

  • L0. 寄存器 字
  • L1. 高速缓存(SRAM) 行
  • L2. 高速缓存(SRAM) 行
  • L3. 高速缓存(SRAM) 行
  • L4. 主存(DRAM) 行
  • L5. 本地二级存储(磁盘) 磁盘块
  • L6. 远程二级存储(Web) 文件

四.操作系统管理硬件

1.进程

进程是操作系统对一个正在运行的程序一种抽象。单核系统中多个进程的指令是交错执行的，形成并行的假象。
上下文切换：这种交错执行的机制称之为上下文切换（通过处理器在进程间切换实现）
上下文：操作系统保持跟踪进程所需要的所有状态信息。这种状态，也就是上下文。
内核：操作系统管理全部进程所用代码和数据结构的集合。

2.线程

一个进程可以由多个称为线程的执行单元组成，线程运行在进程的上下文，并共享同样的代码和全局数据。
线程间的数据共享比进程间的数据共享更加容易和高效。

3.虚拟内存

虚拟内存是一个抽象概念，为每个进程提供一个假象，即每个进程都在独占使用主存。
虚拟地址空间：每个进程看到的内存空间都是一致的。
进程的虚拟地址空间：

  1. 内核虚拟空间 |地址空间顶部，为内核保留。用户代码不可见，必须通过调用内核执行。
  2. 栈(用户栈) |用户虚拟地址空间顶部。运行时创建，编译器用其实现函数调用。可以动态扩展收缩。
  3. 共享库的内存映射区域 |地址空间中间部分。存放C标准库和数学库。
  4. 运行时堆 |代码和数据区之上，在运行时由malloc创建，一开始运行就被指定了大小，可以动态扩展收缩。
  5. 读/写数据 |从文件中加载
  6. 只读的代码和数据 |从文件中加载

    4.文件

    文件就是文件序列。每个I/O设备（磁盘，键盘，显示器，网路）都是文件。系统的所有I/O操作都是通过Unix I/O系统函数调用读写文件来实现。

五.重要概念

1.Amdahl定律

思想：当我们对系统的某个部分加速时，其对系统整体性能的影响取决于该部分的重要性和加速程度。
设:T(old)=系统执行应用程序需要的时间。a=系统某部分需要执行时间和该时间的比例，该部分性能提升比例=k，则该部分初始化所需要时间=aT(old)，现在所需要时间为(aT(old))/k,则总执行时间为：

T(new)=(1-a)T(old) + (aT(old))/k = T(old)[(1-a)+a/k]

所以计算加速比S=T(old)/T(new)为

S=1/[(1-a)+a/k]

主要观点：要想显著加速整个系统，必须提升全系统中相当大的部分的速度。

练习题1: S=1/(1-0.6)+(0.6/1.5)=1.25
练习题2: 1.67=1/（1-0.6)+(0.6/x) 解得x=3.必须以三倍速度(300km/h)
练习题3: 2=1/(1-0.8+0.8/x) 解得x=2.67

特殊情况：Amdahl定律有一个特殊情况是考虑k趋向正无穷时的效果。这意味着，我们可以取系统的某一部分加速到一个点，在这个点上，花费时间可以忽略不计。则

S=1/(1-a)

适用范围：加速计算机系统，公司降低刀片制造成本，学生提高绩点平均值等。

2.并发和并行

并发：一个同时具有多个活动的系统。
并行：用并发来使一个系统运行得更快。

(1).线程级并发:在一个进程执行多个控制流

超线程：也称为同时多线程，一项允许一个CPU执行多个控制流的技术。涉及CPU的某些硬件有多个备份，如程序计数器和寄存器文件，而其他的硬件只有一份，如浮点算术运算单元。
举例：i7处理器可以让每个核执行两个线程，那么4核系统实际上可以并行执行8个线程。

(2).指令级并行:现代处理器可以同时执行多条指令

超标量：处理器可以达到比一个周期一条指令更快的执行速率。

(3).单指令，多数据并行：允许一条指令产生多个可以并行执行的操作。（SIMD并行）

为了提高处理影响，声音，视频是数据应用的执行速度。最近的处理器都具有并行对8对单精度浮点数（C数据结构float)做加法的指令。

3.抽象的重要性

  • 指令集架构提供了对实际处理器硬件的抽象。
    操作系统提供了三个基本抽象：
  1. 文件是对I/O设备的抽象。
  2. 虚拟内存是对程序存储器的抽象。
  3. 进程是对一个正在运行程序的抽象。
  • 虚拟机是对整个计算机的抽象。

视频笔记:

Ints不是整数,Floats不是实数

  • int进行四则运算时，符合交换律结合律，但当结果超过32bit时，数字会溢出。即使是溢出，结果也是符合交换律。
  • float进行四则运算时，结果不会溢出，但是不符合交换律和结合律，且有舍入问题，会舍弃一些数字。（同样会溢出）
    他们都用有限的位组合形式表示在数域中无线扩张的数

内存很重要

  • c不会检查数组边界，操作系统会。
  • 当在c中使用a[1000]访问一个初始化为a[2]的数组时，程序不会报错，而是修改了其他程序/其他变量的值。（在内存上是a[2]后面的第998位。）（最恐怖的噩梦：调试内存引用的错误，因为它可能在修改后的几天甚至更久后才会出现错误，也可能马上就出现错误）
  • 双层数组遍历时，行优先遍历比列优先遍历的效率更高。因为在内存上，列优先遍历是遍历每个1，然后遍历每个2[{1,2,3….1024},{1,2,3…1024}….].而行优先则是以自然的递进顺序1,2,3….（数组是顺序结构，在内存上是顺序的）
