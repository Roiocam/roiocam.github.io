Java在字符串池使用的搜索算法

Posted by Andy Chen on 2019-07-01

一、字符串

字符串的生成

由于Java中字符串的不变性，JVM可以通过在池中存储每个文字字符串的一个副本来优化他们的内存分配。

  1. 当我们创建String 变量并赋值时，JVM会先在池中搜索相同值的String。如果存在，则简单返回对其内存地址的引用，若无，则添加该String到池中，并返回。

  2. 当我们使用new 生成一个String对象时，编译器会创建一个新的对象，并将其存储在JAVA的Heap Space（堆空间）。使用new 创建的字符串都将指向不同的内存地址。

两者的不同是，前者创建的String对象，最终都会被Intern。

垃圾收集

在Java7之前，JVM将String Pool放在PermGen空间，该空间大小固定，无法在运行时扩展，不符合垃圾回收的条件。在PermGen中实现字符串常量池的风险是，如果我们Intern太多字符，我们会得到Out Of Memory错误。

从Java7开始，字符串常量池被存储在Heap 空间，能够被JVM垃圾时回收，这样好处时未引用的字符串将会被在池中删除，从而释放内存，减少OOM的几率。

性能优化

通过调节String Table Size（常量池大小）可以调节程序的性能。

二、字符串常量池

简介

String interning 是一种只存储不同String Value的方法。String在Java中是Immutable (永恒不变的)。因此，这种共享是完全公开和安全的，并且可以提供更好的性能。

不同的值存储在固定大小的HashTable（哈希表）中，通常被叫做字符串Intern池或字符串池。

Intern字符表作为固定大小的哈希表存储在本地（Native）内存中而非堆内存中。

在Java 7u40版本之前，默认表的大小为1009个桶（bucket），在64位的7u40版本及之后，默认的大小为60013. 表的大小会根据预期的数据调整，在java7之后，这个表的大小可以在启动JVM时自定义：

当一个应用需要使用大量的String Intern（字符串常量）时，应该该值调大，并且保证该值永远为质数，以此获得更好的性能。（哈希表内数字的大小）

理想的桶大小

1. 什么是Bucket（存储桶）

哈希表数据结构中，有一个可以容纳指定数量的数组，数组中每个元素称之为存储桶。简单来说，使用散列算法计算桶的位置。两个不同的元素可以有相同的散列值，这种情况成为哈希碰撞（Hash Collision）。通常通过在每个阵列位置（存储桶）创建一个链表，并将散列到该存储桶的每个元素添加到链表来处理冲突。

2.理想的桶大小

理想的情况下，平均桶的大小应该为0或者1（链表的长度），这意味着没有冲突，因性能更好，如果平均值大于1，我们可以相应的增加字符串表的大小（减少哈希碰撞，增加存储哈希桶的数组大小）。每个桶只需要4/8个字节，因此将字符串表的大小设置得太高也不会有明显的性能损失。

总结

总而言之，大量（经常）使用重复的字符串（String）会有利于程序Interning String（将字符串放入常量池）。

intern多个字符串的应用程序可能需要相应地调整字符串intern表的大小。
